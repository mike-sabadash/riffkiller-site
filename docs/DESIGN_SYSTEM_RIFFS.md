# Дизайн-система: лендинг ↔ профиль → страница риффов

Свод правил, по которым выровнена **страница риффов** (`app.html` + `css/riffs-hub.css`).

## Фон и атмосфера

- **База:** чистый чёрный `#000000`, не серо-синий градиент из старого `base.css` на этой странице.
- **Ambient:** три мягких радиальных блика (как `landing.html` / `profile-landing-sync.css`): серо-голубой слева, бирюзовый справа сверху, фиолетовый снизу справа (`rgba(147, 85, 229, …)`).
- Контент в `.page-container` с **`z-index: 1`** поверх псевдо-фона.

## Типографика

- **UI / body:** `Inter` (variable opsz 14–32), сглаживание как на лендинге.
- **Один display-уровень (hero):** только заголовок страницы — `TT Travels Next` / Space Grotesk, ~600, как крупный h1 на landing.
- **Секции (Collections, Most killed, и т.д.):** `Inter` 600, **без TT** — как h2/h3 на landing (`section-title` там тоже через Inter для подзаголовков секций).
- **Лид и описания:** Inter 400, вторичный цвет `rgba(160, 165, 182, …)`, `line-height` ~1.55–1.6.
- **Метки / eyebrow / фильтры:** Inter 600, uppercase, умеренный `letter-spacing`.
- **Карточки (название трека):** Inter 600 ~0.875rem; **артист:** Inter 400 ~0.8125rem.

## Цвета и акценты

- **Primary CTA / акцент:** `#9355E5` (как кнопки на лендинге и шапке).
- **Границы плашек:** `rgba(255, 255, 255, 0.06–0.12)`.
- **Стекло:** `background: rgba(255, 255, 255, 0.03–0.06)` + `backdrop-filter: blur(16–24px)` на панелях инструментов.
- **Сложность риффа:** beginner — зелёный спектр, intermediate — янтарь, advanced — красно-коралловый (как семантика, не «материал Android»).

## Компоненты

- **Панель фильтров / шапка блока коллекций:** одна скруглённая «стеклянная» полоса (`border-radius: 16px`), внутри — заголовок, селекты, стрелки.
- **Селекты:** высота ~40px, радиус `12px`, бордер `1px solid rgba(255,255,255,0.12)`, без тяжёлого `#7C7C7C`.
- **Стрелки карусели:** одна обводка (только SVG-круг), hover — лёгкий фон, без второго border на кнопке.
- **Карточки коллекций / риффов:** `border-radius: 16px`, тонкая обводка, тень при hover с фиолетовым тинтом; без резкого скачка только по `translateY`.
- **FREE / бейджи:** компактные, в духе UI лендинга (меньше «плашка Material»).

## Сетка и отступы

- **Max-width контента:** `1240px` (как `.container` на лендинге), горизонтальные отступы `28px` → на мобиле `20px`.
- **Секции:** увеличенный вертикальный ритм между hero → collections → riffs.
- **Most killed riffs:** одна fluid-сетка `repeat(auto-fill, minmax(min(100%, --riffs-hub-tile-min), 1fr))` и **единый `gap`** (`--spacing-lg`, как у `.collections-grid`). Карточки **на всю ширину ячейки** (`width: 100%`, без `max-width` + `justify-items: start`), чтобы не было «дырок» и скачков при смене брейкпоинтов.

## Две галереи на одном экране (группы + одиночные элементы)

Паттерны из продуктов вроде **Netflix / Apple Music / Spotify**:

1. **Верх — горизонтальная полка (shelf)** для *групп* (коллекции): фиксированная ширина карточки, `overflow-x: auto`, стрелки или свайп — ментальная модель «смотри категории, зайди внутрь».
2. **Ниже — сетка или вторая полка** для *атомарных* элементов (риффы): либо **та же горизонтальная полка** (если нужен единый жест скролла), либо **responsive grid** с равными ячейками — ментальная модель «обзор плиток одного типа».
3. **Согласованность:** одинаковый **aspect ratio** карточки (у нас **5:3** как у коллекции) и **одинаковый вертикальный/горизонтальный gap** между полкой и сеткой не даёт ощущения «две разные страницы».
4. **Избегать:** смешивать `repeat(N, 1fr)` на разных брейкпоинтах с другим базовым `auto-fill` — получаются разные пробелы и «ломаная» плитка; лучше **одна формула** + при необходимости только CSS-переменная `tile-min` на очень узких экранах.
5. **Опционально (идея на будущее):** вынести «Most killed» в **ту же горизонтальную полку**, что и коллекции, если нужен единственный паттерн скролла; либо **вкладки** «Collections | Riffs» с одной сеткой ниже — меньше визуального шума на маленьком экране.
